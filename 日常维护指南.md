## 一、日常开发流程

### 改代码（本地电脑操作）
```bash
cd E:\yiyou_factory
# 改完代码后
git add .
git commit -m "简短说明改了什么"
git push
```

### 部署到服务器
```bash
ssh root@101.43.91.152      # 或者你自己的登录方式
su - yiyou
cd /home/yiyou/yiyou_factory
git pull
```

如果改了 Python 依赖（requirements.txt）：
```bash
source venv/bin/activate
pip install -r requirements.txt
```

如果改了数据库表结构（models.py）：
```bash
source venv/bin/activate
flask db migrate -m "说明改了什么"
flask db upgrade
```

最后重启服务：
```bash
sudo systemctl restart yiyou
```

### 验证服务正常
```bash
sudo systemctl status yiyou
```
看到 active (running) 就是正常的。浏览器访问系统确认能登录。

## 二、备份相关

### 查看备份文件
```bash
ls -lh /home/yiyou/backups/
```
正常情况下每天凌晨3点会多一个新文件，格式为 `yiyou_factory_YYYYMMDD_0300.sql`。

### 手动立刻备份一次
```bash
bash /home/yiyou/backup_db.sh
```

### 验证备份是否有效
```bash
mysql -u root -p
CREATE DATABASE yiyou_test;
GRANT ALL ON yiyou_test.* TO 'yiyou'@'localhost';
exit;

mysql -u yiyou -p yiyou_test < /home/yiyou/backups/最新的那个文件名.sql
mysql -u yiyou -p yiyou_test
SHOW TABLES;
SELECT COUNT(*) FROM customers;
exit;

# 验证完删掉
mysql -u root -p
DROP DATABASE yiyou_test;
exit;
```

### 从备份恢复（真正出事时用）
```bash
# 先停服务
sudo systemctl stop yiyou

# 恢复数据库（会覆盖当前所有数据）
mysql -u yiyou -p yiyou_factory < /home/yiyou/backups/要恢复的文件名.sql

# 重启服务
sudo systemctl restart yiyou
```
⚠️ 恢复会覆盖当前数据库，操作前确认你选的备份文件是对的。

## 四、服务异常排查

### 网站打不开
按顺序检查：

```bash
# 1. 看应用是否在运行
sudo systemctl status yiyou

# 2. 如果没在运行，看报错原因
sudo journalctl -u yiyou --no-pager -n 50

# 3. 看 Nginx 是否正常
sudo systemctl status nginx

# 4. 看端口是否在监听
sudo ss -tlnp | grep 8000
```

### 常见问题及解决

**服务启动失败，提示 SECRET_KEY 或 DATABASE_URL 未设置：**
```bash
cat /home/yiyou/yiyou_factory/.env
```
检查 .env 文件内容是否正确。

**页面报500错误：**
```bash
sudo journalctl -u yiyou --no-pager -n 100
```
看最近的日志找具体报错。

**数据库连接失败：**
```bash
sudo systemctl status mysql
```
如果 MySQL 没运行：`sudo systemctl start mysql`

## 七、操作习惯提醒

1. **改代码永远在本地改**，不要SSH到服务器直接改
2. **每次改完一个功能就 commit 一次**，不要攒一堆再提交
3. **部署前先在本地看一遍改动**：`git diff` 看改了什么
4. **重大改动前先手动备份一次数据库**：`bash /home/yiyou/backup_db.sh`
5. **每周看一眼备份目录**，确认自动备份在正常跑
6. **试用期内发现问题记到问题清单**，不要马上改代码
